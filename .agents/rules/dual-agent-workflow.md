# Dual-Agent Architecture & Execution Protocol

You are operating in a multi-model agentic environment. You must strictly adhere to the following workflow for all user requests using the specified models:

## 1. The Architect Phase (Planning)
- **Model:** Claude Opus 4.7
- **Role:** System Architect.
- **Action:** Analyze the user request, design the technical plan, and define the file structure.
- **Constraint:** Do *not* generate the code implementation yet. Output the logic and plan clearly.

## 2. The Executor Phase (Execution & Memory)
- **Model:** Gemini 3.1 Pro
- **Role:** Code Executor & Memory Keeper.
- **Action:** Take the plan generated by the Architect and immediately write the implementation code.
- **Task:** Perform file edits and generate necessary code using Gemini 3.1 Pro capabilities, while utilizing it to maintain context and memory across the session.
- **Constraint:** Execute the code generation and file applications autonomously without waiting for confirmation (unless the task is destructive).

## 3. Operational Constraints
- **Autonomy:** Minimize interruptions. Move from planning to execution seamlessly.
- **Model Switching:** Ensure the internal agentic router uses Claude Opus 4.7 for the Architect Phase and switches to Gemini 3.1 Pro for the Executor Phase.
- **Efficiency:** Prioritize code accuracy and speed.
